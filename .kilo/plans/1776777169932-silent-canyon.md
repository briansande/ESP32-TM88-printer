# Refactoring Plan: Decompose `main.cpp` God Object

## Summary

Split the monolithic 899-line `src/main.cpp` into 4 focused modules under `include/` (headers) and `src/` (implementations), following PlatformIO conventions. Zero behavioral changes.

## New File Structure

```
include/
  printer_driver.hpp   — PrinterDriver class (ESC/POS hardware abstraction)
  base64.hpp           — Base64 decode utility (free functions)
  web_server.hpp       — PrinterWebServer class (HTTP routes + embedded HTML)
  wifi_secrets.h       — (unchanged)
src/
  printer_driver.cpp   — PrinterDriver implementation
  base64.cpp           — Base64 implementation
  web_server.cpp       — PrinterWebServer implementation
  main.cpp             — Entry point: WiFi connect, instantiate objects, run loop
```

## Module Design

### 1. `PrinterDriver` (`printer_driver.hpp/.cpp`)

**Responsibility:** All ESC/POS serial communication with the thermal printer.

**Class interface:**
```cpp
class PrinterDriver {
public:
  PrinterDriver(int rx, int tx, int baud);
  void begin();           // init serial + reset
  void reset();           // ESC @
  void print(const String& text);
  void cut();
  void feed(uint8_t n = 5);
  int getStatus();        // returns raw byte or -1 on timeout

  void setBold(bool on);
  void setUnderline(bool on);
  void setDoubleHeight(bool on);
  void setReverse(bool on);

  void justifyLeft();
  void justifyCenter();
  void justifyRight();

  void setLineSpacing(uint8_t n);
  void defaultLineSpacing();
  void setCharacterSet(uint8_t n);

  void setBarcodeHeight(uint8_t n);
  void setBarcodeWidth(uint8_t n);
  void setBarcodeNumberPosition(uint8_t n);
  void printBarcode(uint8_t m, const String& data);

  void printImage(uint16_t widthDots, uint16_t heightDots,
                  const uint8_t* data, size_t dataLen);

private:
  HardwareSerial _serial;
  int _rx, _tx, _baud;
  uint8_t _style;         // current ESC/POS style byte
  void applyStyle();
};
```

**Key decisions:**
- Uses `HardwareSerial` member (constructed with `2` for UART2, matching original).
- `_style` byte tracks bold/underline/doubleHeight state internally (matches `currentStyle` global).
- `begin()` must be called in `setup()` after Serial is ready.

### 2. `Base64` (`base64.hpp/.cpp`)

**Responsibility:** Base64 decoding utility. Stateless free functions.

```cpp
namespace base64 {
  int charVal(char c);
  size_t decode(const char* in, size_t inLen, uint8_t* out);
}
```

Wrapped in a namespace to avoid symbol collisions. Logic identical to original.

### 3. `PrinterWebServer` (`web_server.hpp/.cpp`)

**Responsibility:** HTTP server with embedded HTML UI and route handlers.

```cpp
class PrinterWebServer {
public:
  PrinterWebServer(PrinterDriver& printer);
  void begin();

private:
  WebServer _server;
  PrinterDriver& _printer;
  static const char HTML_PAGE[];  // PROGMEM raw literal

  void registerRoutes();
  void handleRoot();
  void handlePrint();
  void handleCut();
  void handleFeed();
  void handleStatus();
  void handleBold();
  void handleUnderline();
  void handleDoubleHeight();
  void handleReverse();
  void handleJustify();
  void handleLineSpacing();
  void handleDefaultLineSpacing();
  void handleCharacterSet();
  void handleBarcode();
  void handlePrintImage();
};
```

**Key decisions:**
- Takes `PrinterDriver&` — delegates all printer ops to it.
- `HTML_PAGE` stays as a `static const char[] PROGMEM` — exact same HTML string.
- `handleClient()` is NOT wrapped — main loop calls `_server.handleClient()` ... actually, we need to expose that. I'll add a `handleClient()` passthrough method so `loop()` stays clean.

Actually, simpler approach: expose `loop()` method on PrinterWebServer that just calls `_server.handleClient()`. This keeps the main loop minimal.

### 4. Refactored `main.cpp`

```cpp
#include <Arduino.h>
#include <WiFi.h>
#include "wifi_secrets.h"
#include "printer_driver.hpp"
#include "web_server.hpp"

static PrinterDriver printer(16, 17, 19200);
static PrinterWebServer webServer(printer);

void setup() {
  Serial.begin(115200);
  delay(500);
  printer.begin();

  WiFi.mode(WIFI_STA);
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) delay(500);

  webServer.begin();
}

void loop() {
  webServer.loop();
}
```

## Implementation Steps

1. Create `include/base64.hpp` and `src/base64.cpp`
2. Create `include/printer_driver.hpp` and `src/printer_driver.cpp`
3. Create `include/web_server.hpp` and `src/web_server.cpp`
4. Rewrite `src/main.cpp` to slim entry point
5. Build with `pio run` to verify compilation

## Constraints

- All functional logic, timings, and algorithms remain byte-for-byte identical
- No new library dependencies
- `wifi_secrets.h` unchanged
- PlatformIO build system (`platformio.ini`) unchanged — it auto-compiles all `.cpp` in `src/` and includes `include/` in the path
- All headers use `#pragma once`
