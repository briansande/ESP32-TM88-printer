# Fix: Barcode not printing on TM88

## Root Cause

`printBarcode()` in `src/printer_driver.cpp:155-163` is missing a line feed (0x0A) after the barcode command. Unlike `printImage()` which flushes and feeds after sending data, `printBarcode()` sends the GS k command and returns immediately — the barcode stays buffered and never makes it to paper.

## Fix

### 1. Add flush + line feed to `printBarcode()` (`src/printer_driver.cpp`)

After the barcode data loop, add:
```cpp
_serial.flush();
_serial.write(0x0A);
```

This matches the pattern used by `printImage()` at line 175-176.

### 2. Set default barcode parameters before printing (`src/printer_driver.cpp`)

Before the GS k command, configure sensible defaults:
- Barcode height: `GS h 0xA0` (160 dots, visible)
- Barcode width: `GS w 0x03` (module width 3, medium)
- HRI position: `GS H 0x02` (text below barcode)

These ensure the barcode is visible even after a fresh printer reset.
