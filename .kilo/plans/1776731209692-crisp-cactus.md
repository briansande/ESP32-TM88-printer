# Plan: Add Image Printing to ESC/POS Printer

## Overview

Add image printing capability using the **GS v 0** (Print Raster Bit Image) ESC/POS command. The approach uses **client-side dithering** (JavaScript in the browser) to minimize ESP32 processing and memory requirements. The ESP32 receives a pre-dithered 1-bit bitmap and sends it directly to the printer.

## Technical Details

### ESC/POS Command: GS v 0

```
Byte sequence: 0x1D 0x76 0x30 m xL xH yL yH d1...dk

m  = 0x00 (normal size) or 0x01 (double width/height)
xL = (widthInBytes) & 0xFF       // width in BYTES = ceil(pixelWidth / 8)
xH = (widthInBytes >> 8) & 0xFF
yL = heightInDots & 0xFF
yH = (heightInDots >> 8) & 0xFF
d  = row-major 1-bit bitmap, MSB = leftmost pixel, 1 = black (fire dot)
```

### Why GS v 0 over ESC *

| Factor | GS v 0 | ESC * |
|--------|--------|-------|
| Data format | Row-major (natural) | Column-major (complex) |
| Code complexity | Simple (one command) | Complex (strip loops) |
| Epson TM support | Excellent | Universal |
| Memory | Must buffer all | Can stream strips |

GS v 0 is chosen for simplicity and excellent Epson TM-series support.

### Memory Budget (ESP32)

- Max image: 576px wide x 800px tall
- 1-bit bitmap: `ceil(576/8) * 800 = 57,600 bytes`
- Base64 string: ~76,800 bytes
- Peak during decode: ~134KB (both buffers exist briefly)
- ESP32 available heap: ~320KB — feasible

## Implementation

### 1. C++ Changes (src/main.cpp)

#### a) Base64 decoder utility

A lookup-table base64 decoder that decodes a String into a pre-allocated buffer:

```cpp
static int base64CharVal(char c);
static size_t base64Decode(const char* in, size_t inLen, uint8_t* out);
```

#### b) Image printing function

```cpp
static void printerPrintImage(uint16_t widthDots, uint16_t heightDots, const uint8_t* data, size_t dataLen) {
    uint16_t widthBytes = (widthDots + 7) / 8;
    // Send GS v 0
    printer.write(0x1D); printer.write(0x76); printer.write(0x30);
    printer.write(0x00);  // normal size
    printer.write(widthBytes & 0xFF);
    printer.write((widthBytes >> 8) & 0xFF);
    printer.write(heightDots & 0xFF);
    printer.write((heightDots >> 8) & 0xFF);
    printer.write(data, dataLen);
    printer.write(0x0A);  // feed one line
}
```

#### c) HTTP handler `handlePrintImage()`

- Reads `width`, `height`, and `data` (base64) from POST form params
- Validates dimensions (width <= 576, height <= 800)
- Decodes base64 into heap-allocated buffer
- Calls `printerPrintImage()`
- Frees buffer
- Returns JSON `{"ok": true/false}`

#### d) Register route

```cpp
server.on("/printImage", HTTP_POST, handlePrintImage);
```

### 2. HTML/JavaScript Changes (HTML_PAGE in main.cpp)

#### a) New UI section: "Image Print"

Added between the Barcode section and the Status button:

- **File input** for selecting an image
- **Paper width selector**: 58mm (384 dots) / 80mm (576 dots)
- **Dithering method selector**: Floyd-Steinberg / Threshold
- **Preview canvas** showing the dithered result
- **Print Image** button

#### b) JavaScript image processing pipeline

1. **File selection** → read with `FileReader` as data URL
2. **Draw to offscreen canvas** scaled to printer width, maintaining aspect ratio
3. **Get `ImageData`** from canvas
4. **Convert to grayscale**: `gray = R*0.299 + G*0.587 + B*0.114`
5. **Apply dithering**:
   - **Floyd-Steinberg** (default): error diffusion to 4 neighbors (7/16, 3/16, 5/16, 1/16)
   - **Threshold**: simple `gray < 128 ? black : white`
6. **Pack 1-bit data** into `Uint8Array` (row-major, MSB = leftmost, 1 = black)
7. **Encode as base64** using `btoa(String.fromCharCode.apply(null, bytes))`
8. **Show preview** on a visible canvas
9. **On Print**: POST to `/printImage` with `width`, `height`, and `data` params

#### c) Floyd-Steinberg dithering implementation

Uses a `Float32Array` grayscale buffer for error diffusion precision:

```
For each pixel (left→right, top→bottom):
  newVal = oldVal < 128 ? 0 : 255
  error = oldVal - newVal
  distribute error: right 7/16, below-left 3/16, below 5/16, below-right 1/16
```
