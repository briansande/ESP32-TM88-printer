# Fix Barcode and Image Printing

## Problem Analysis

### Issue 1: Barcode not printing, first character cut off

**Root cause:** `printBarcode()` in `printer_driver.cpp:155-169` uses ESC/POS "Function B" format (`GS k m n d1...dn` with m=65-73, length-prefixed). Many thermal printers (including some TM-T88 variants and most clones) only support "Function A" format (`GS k m d1...dk NUL` with m=0-6, NUL-terminated).

When a Function-A-only printer receives Function B data:
1. `m=69` (CODE39) is out of the valid range (0-6), so the command is rejected/misinterpreted
2. The length byte (`data.length()`) gets consumed as part of the failed command parsing
3. The first data character is also consumed or shifted
4. Remaining bytes are output as plain text — no barcode rendered, first char missing

**Evidence:** The "first character cut off" symptom perfectly matches a Function-A printer consuming the length byte as command data, then misinterpreting the first actual data byte.

### Issue 2: Image not printing

**Root cause:** Likely the image data being sent to the printer too fast. The `_serial.write(data, dataLen)` call at `printer_driver.cpp:180` sends all image data at once. At 19200 baud, a 576x200 image is 14,400 bytes — the printer's small internal receive buffer can overflow, causing data loss. May also be compounded by ESP32 memory pressure from large base64 POST bodies.

## Fixes

### Fix 1: Switch barcode to Function A format

**File: `src/printer_driver.cpp` — `printBarcode()` (line 155)**

- Replace Function B format (m=65-73, length-prefixed) with Function A (m=0-6, NUL-terminated)
- Auto-prepend/append `*` delimiters for CODE39 (m=4) if not already present
- Change: `_serial.write((uint8_t)data.length())` → `_serial.write(0x00)` (NUL terminator after data)

```cpp
void PrinterDriver::printBarcode(uint8_t m, const String& data) {
  setBarcodeHeight(162);
  setBarcodeWidth(3);
  setBarcodeNumberPosition(0x02);

  String barcodeData = data;
  if (m == 4 && !barcodeData.startsWith("*")) {
    barcodeData = "*" + barcodeData + "*";
  }

  _serial.write(0x1D);
  _serial.write(0x6B);
  _serial.write(m);
  for (unsigned int i = 0; i < barcodeData.length(); i++) {
    _serial.write(barcodeData[i]);
  }
  _serial.write(0x00);
  _serial.flush();
  _serial.write(0x0A);
}
```

**File: `src/web_server.cpp` — HTML barcode options (line 168-178)**

Update `<option>` values from 65-73 to 0-6. Remove CODE93 (72) and CODE128 (73) since they're only available in Function B.

### Fix 2: Send image data in chunks with delays

**File: `src/printer_driver.cpp` — `printImage()` (line 171-183)**

Send image data line-by-line with `_serial.flush()` and a small delay between chunks to prevent the printer's receive buffer from overflowing.

```cpp
void PrinterDriver::printImage(uint16_t widthDots, uint16_t heightDots,
                                const uint8_t* data, size_t dataLen) {
  uint16_t widthBytes = (widthDots + 7) / 8;
  _serial.write(0x1D); _serial.write(0x76); _serial.write(0x30);
  _serial.write(0x00);
  _serial.write(widthBytes & 0xFF);
  _serial.write((widthBytes >> 8) & 0xFF);
  _serial.write(heightDots & 0xFF);
  _serial.write((heightDots >> 8) & 0xFF);

  size_t sent = 0;
  while (sent < dataLen) {
    size_t chunk = (dataLen - sent > 240) ? 240 : (dataLen - sent);
    _serial.write(data + sent, chunk);
    sent += chunk;
    _serial.flush();
    delay(10);
  }
  _serial.write(0x0A);
}
```
